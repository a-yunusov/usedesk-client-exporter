# Usedesk Client Exporter

Консольное приложение на .NET 8 для выгрузки данных о клиентах из Usedesk через API и сохранения их в CSV-файл.

## Описание

Скрипт экспортирует информацию о клиентах из системы Usedesk:
- ID клиента
- Имя
- Email-адреса
- Телефоны
- Номера тикетов

Выгружаются только клиенты, у которых указан email. Данные сохраняются в файл `clients.csv` в формате CSV с разделителем `;`.

## Получение секретного ключа

1. Откройте настройки Usedesk
2. Перейдите в раздел **Каналы**
3. Создайте новый **API-канал**
4. Скопируйте сгенерированный секретный ключ

## Использование

1. Запустите приложение
2. Введите секретный ключ API при запросе
3. Дождитесь завершения экспорта
4. Результат будет сохранён в файле `clients.csv`

## Формат выходного файла

Файл `clients.csv` содержит следующие колонки:

| Колонка   | Описание                      |
|-----------|-------------------------------|
| ID        | Уникальный идентификатор      |
| Name      | Имя клиента                   |
| Emails    | Список email-адресов          |
| Phone     | Список телефонов              |
| TicketIDs | Номера связанных тикетов      |

Пример:
```csv
ID;Name;Emails;Phone;TicketIDs
12345;"Иван Петров";"ivan@example.com";"+79991234567";1001,1002
```

## Настройка

### Параметры в `Program.cs`

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `limit` | Количество клиентов на странице (1–100) | 100 |
| Фильтр по email | Пропуск клиентов без email | Включён |
| Формат CSV | Колонки для экспорта | ID, Name, Emails, Phone, TicketIDs |

### Параметры в `UseDeskClientService.cs`

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `MaxRetries` | Максимальное число повторных попыток запроса | 5 |
| Задержка при 429 | Пауза при Too Many Requests | 120 секунд |
| Задержка при других ошибках | Пауза перед повторной попыткой | 60 секунд |

## Обработка ошибок

Приложение автоматически обрабатывает следующие ситуации:

- **Too Many Requests (429)** — ожидание 120 секунд и повтор запроса
- **Другие HTTP-ошибки** — ожидание 60 секунд и повтор запроса
- **Невалидный JSON** — логирование фрагмента ответа и повтор запроса
- **Сетевые ошибки** — повтор запроса после 1 секунды

После исчерпания попыток (по умолчанию 5) приложение продолжает работу со следующим клиентом.

## API Usedesk

Приложение использует следующие эндпоинты:

- `GET /clients` — получение списка клиентов с пагинацией
- `GET /client` — получение детальной информации о клиенте

Документация: https://api.usedocs.ru/
